# 完整产品能力

DataLuminary 的完整产品能力围绕 **数据源 → 数据集 → 指标治理 → 分析创作 → 可视化表达 → AI 洞察 → 权限协作 → 私有化交付** 展开。白皮书文档描述产品应呈现给用户的完整形态；具体迭代、验收和工程拆解维护在 `plan/` 与 `spec/` 中。

## 账号与权限

- 用户注册、登录、本地账号与企业 SSO。
- 组织、空间、项目、资源等多层级协作模型。
- 基于角色、资源、数据范围和操作权限的访问控制。
- 审计、版本、发布、订阅与访问统计。

## 数据接入

- 数据库连接：MySQL、PostgreSQL、ClickHouse、SQL Server 等。
- 文件导入：Excel、CSV 与结构化文件。
- API 与业务系统连接器：REST、GraphQL、内部服务和第三方 SaaS。
- IoT、日志、Agent 运行数据等事件与遥测数据接入。
- 数据源测试、凭据管理、连接状态与查询预览。

## 数据集

- 基于数据源创建可复用数据集。
- 配置指标、维度、字段类型、过滤、聚合和计算口径。
- 支持拖拽建模、SQL 高级模式和 AI 辅助建模。
- 提供字段字典、指标说明、血缘关系、数据质量提示和预览查询。
- 数据集作为图表、仪表盘、报表、API 和 AI 洞察的统一语义入口。

## 图表与可视化

- **NoCode 配图**：通过配置界面选择图表类型与数据映射。
- **AI 配图**：根据分析目标推荐图表、字段映射和视觉表达。
- **图表类型**，例如：
  - 折线、柱状、面积及堆叠/组合变体
  - 饼图、环形、玫瑰、雷达
  - 表格、散点、漏斗、词云、地图
  - 指标卡、仪表盘、水球、富文本、图片、视频等
- 图表作为资源管理，可挂载到仪表盘、报表、大屏和嵌入式场景。
- 图表配置沉淀为平台原生资产，可被 AI 生成、修改和复用。

## 仪表盘与大屏交互

- 仪表盘创建、编辑、发布、订阅与管理。
- 网格布局、自由布局、列表布局等多形态面板（见 [布局选型](#仪表盘布局选型)）。
- 目录分类、版本记录、收藏、访问统计和分享控制。
- 交互组件：输入、下拉、单选、多选、时间选择、变量、联动、下钻。
- 管理驾驶舱、经营看板、生产大屏、移动看板、审计报表等多场景输出。

### 状态驱动的高性能交互引擎

DataLuminary 的图表联动与交互，采用 **状态驱动 (State-Driven) + 派生计算** 架构，而非全局共享过滤大对象或图表间事件互抛：

- **操作即数据**：用户的每一次筛选、图表点击、下钻，都直接化为仪表盘的内存状态。
- **动态派生图表条件**：无需写代码配置互相监听，过滤计算引擎会根据配置智能、实时地推导出各个图表应该附加的数据条件。
- **极致的渲染性能**：只有当一个图表确切需要新数据时（其附加条件真正改变时），才会独立进行请求和重绘，其他图表完全静默。这意味着即便画布上放置上百张图表，错综复杂的联动也不会导致卡顿。
- **URL 级别的分享穿透**：所有交互状态高度序列化，可以直接挂载到 URL 上。用户分享某个仪表盘链接给同事，同事打开后看到的数据范围、下钻层级和联动状态，与分享时完全一致。

架构对比与实现细节见 [设计与实现](../develop/dashboard-interact-engine.md)。

### 仪表盘布局选型

| 布局 | 适用场景 | 画布语义 |
|------|----------|----------|
| **自由布局** | 指挥大屏、展厅、拼接屏 | 固定逻辑尺寸（如 1920×1080、4K）+ 整体等比缩放，**一屏看全** |
| **网格布局** | PC 门户、可滚动经营看板 | 宽度铺满，**高度随内容**增长，流式栅格 |
| **列表布局** | 移动端单列 | 固定逻辑宽度 + 列表流 |

自由布局 **不支持**「固定宽度、高度自由撑高」——绝对定位画布无法可靠实现，且与标准大屏体验冲突。详见 [自由布局大屏固定尺寸](./position-layout-fixed-canvas.md)。

## AI 数据洞察

- 自然语言问数：围绕业务问题查找数据、生成图表和解释结果。
- 分步骤分析：AI 依次辅助数据集、指标、图表、布局和洞察报告。
- 一句话生成：从目标描述生成可编辑、可审计的仪表盘草案。
- 洞察报告：异常检测、趋势解释、归因分析、指标波动说明和行动建议。
- AI 生成结果落在平台原生资源体系内，可继续编辑、发布、审计和二次开发。

## 插件化底座

- 数据源插件、图表插件、面板插件、交互插件、AI 插件分层设计。
- 通过 Schema 化配置承载图表属性、数据映射、面板布局和插件元信息。
- 平台负责资源、权限、配置传导与运行时编排，插件专注具体能力。
- 适合社区扩展新的数据源、行业图表、交互控件、AI Agent 和展示形态。
- 支持插件模板、行业包、主题包和交付方案沉淀。

## 部署与工程

- RsPress 文档站、React 前端、NestJS 后端组成完整产品文档与工程体系。
- 全栈 TypeScript，降低前后端、插件、契约和 AI 辅助开发成本。
- 默认面向私有化部署和企业环境适配，支持从试点到企业级数据规模逐步扩展。
- 支持 Docker 化初始化、PostgreSQL 持久化、备份恢复、升级脚本和可观测性集成。
- 面向客户行业差异，通过插件和模板适配不同业务场景。

## 产品边界说明

- **自由布局大屏**：采用固定逻辑宽高 + 等比缩放，不做「高度随内容撑高」；可滚动分析请用网格布局。详见 [自由布局大屏固定尺寸](./position-layout-fixed-canvas.md)。
- **无障碍（视障 / 读屏）**：主线产品 **不** 适配盲人、弱视用户的无障碍阅读与操作优化；品类为视觉化 BI，详见 [无障碍范围说明](./accessibility-scope.md)。
